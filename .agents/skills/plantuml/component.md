# Component Diagram Conventions

- **Boundaries**: Map architectural layers (UI, Controllers, Services, Repositories).
- **Interfaces**: Expose clear ports and explicitly show provided `()` and required `-->` interfaces.
- **Granularity**: Focus on high-level system modules, frameworks, and external integrations (e.g., IoT Sensors, Database).
- **Deployment Nodes**: If relevant, wrap components in `node` or `cloud` to specify execution environments.
