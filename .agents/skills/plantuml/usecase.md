# Use Case Diagram Conventions

- **Actors**: Clearly separate primary actors (left) and secondary actors/systems (right) using explicit positioning (e.g., `left to right direction`).
- **Relationships**: Use standard `<<include>>` for mandatory dependencies and `<<extend>>` for optional behaviors.
- **Boundary**: Wrap use cases inside a `rectangle "Nome do Sistema"` system boundary to cleanly delineate scope from actors.
- **Simplicity**: Focus purely on the interaction goal, keeping the diagram flat and avoiding deep technical details.
